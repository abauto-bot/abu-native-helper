# ABU Native Helper V9.8 — Accessibility Service Prototype

## Added

- AbuAccessibilityService.kt skeleton
- Accessibility service config XML
- Manifest service declaration
- Service intentionally disabled:
  android:enabled="false"
- Accessibility Settings shortcut
- Safety explanation in app UI

## Important

V9.8 cannot control the phone.

Disabled:
- Tap
- Scroll
- Type
- Screen content read
- Gestures
- Server sync
- Command execution

## Future Requirements

Before Accessibility can be enabled:
1. Explicit user approval
2. Android Accessibility manual permission
3. ABU OS audit event
4. Risk classification
5. Confirmation phrase
6. Emergency stop
7. Strict allowlist
8. Full logs visible to user

## Blocked

- Hidden control
- Silent spying
- Bypass lock screen
- Unauthorized tap/type
- Auto message send
- GOVO destructive actions
- Kasm modification
